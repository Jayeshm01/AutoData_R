'As a general rule (subject to numerous exceptions, caveats, etc.):1) it is programming and debugging time that most impacts 'overall' program execution time;2) this is most strongly impacted by code readability and size (the smaller the better);3) both of which are enhanced by modular construction and reuseability, which argues for avoiding inline code and using separate functions.These days, I would argue that most of the time it is program clarity and correctness (they are related) that is the important issue, not execution speed.'
